# Manifesto

An AI coding agent that forgets everything between sessions is not a collaborator. It is a tool you have to re-teach every time you pick it up. The build command you verified last Tuesday, the architectural constraint you explained three times, the debugging session that narrowed a root cause over forty minutes of careful investigation — all of it evaporates the moment the session ends. The next session starts cold. The agent re-discovers what it already knew. The human repeats what they already said. This is the default behavior of every major coding agent today, and it is quietly wasting enormous amounts of human attention.

The instinctive fix is to give agents more memory. Store everything. Record every file read, every command, every exchange. But storage without discipline produces a different failure mode: an agent that retrieves stale facts with high confidence, that trusts its own records over what the code actually says, that buries a critical constraint under a hundred session transcripts of equal weight. Raw accumulation is not memory. It is hoarding.

The problem is not capacity. The problem is that agents do not understand what memory is for.

Code is the artifact. Memory is commentary on the artifact — what someone understood at the moment they stored it. This distinction determines everything that follows. An agent that treats its memory as authoritative will act on outdated interpretations while the code has already moved on. An agent that ignores memory entirely will start from nothing every session, re-deriving context that was already synthesized. The correct position is neither. Memory is valuable precisely when it knows its own limits: when it searches before assuming, stores conclusions rather than transcripts, yields to the code when they conflict, and decays gracefully when it becomes stale.

From this frame, a set of behaviors follows naturally. Search memory before acting — not because the answer is always there, but because the cost of checking is negligible and the cost of re-deriving is not. Store durable knowledge after investigations — the synthesized conclusion, not the intermediate steps that led to it. Write a handoff at the end of each session so the next one does not start blind. Use structured types so a build command is not stored the same way as an architectural rationale. Let knowledge decay over time — pin only what is truly stable, and let everything else fade unless it is refreshed by continued relevance. When memory and code disagree, trust the code — but when the memory records a deliberate decision, flag the conflict rather than silently overwriting it. Memory gaps are normal. They are not failures. They fill over time.

These are not novel ideas. They are engineering discipline applied to a domain that is now receiving greater attention. Any system that manages state across sessions — caches, databases, event logs — follows equivalent principles. Agent memory should not be an exception.

Signet-first is one implementation of this discipline: a behavioral protocol that enforces these principles at the infrastructure level, across platforms, degrading gracefully when the full memory stack is not available. It is not the only possible implementation. But the principles it encodes — search before assuming, store with precision, trust the artifact, let commentary decay — are not optional. They are what separates an agent that collaborates from one that merely executes.
